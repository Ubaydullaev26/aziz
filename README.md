# Aziz — живая карта города для туристов

Платформа, которая показывает туристу всё интересное вокруг него **прямо
сейчас**: достопримечательности, гидов, экскурсии, события и рестораны одного
города — в одном приложении, в реальном времени. Первый город — Самарканд.

## Стек

- **Frontend/Backend:** Next.js 15 (App Router), React 19, TypeScript (strict),
  Route Handlers + Server Actions
- **UI:** TailwindCSS, самописный shadcn-style UI-кит на Radix UI, Framer Motion
- **База данных:** PostgreSQL + Prisma ORM
- **Авторизация:** Auth.js v5 — Credentials (JWT), Google OAuth, Apple Sign In
- **Карта:** Mapbox GL JS (react-map-gl)
- **Состояние/данные:** Zustand (фильтры карты), TanStack Query (кэш запросов)
- **i18n:** next-intl (ru / en / uz)
- **Инфраструктура:** Docker + docker-compose (Postgres + приложение)

## Архитектура

Проект — модульный монолит на Next.js: один деплой, но код разбит по
feature-модулям (`src/features/*`), а не по типам файлов. Это осознанный
выбор для скорости старта стартапа — но каждый модуль независим настолько,
что букинг, уведомления или платежи можно вынести в отдельный сервис позже,
не переписывая остальное приложение.

Ключевые решения:

- **Категории — это данные, а не код.** Таблица `Category` в БД, а не enum.
  Админ может добавить новую категорию («винная дегустация», «стрит-арт») для
  нового рынка без деплоя.
- **География готова к масштабированию на другие страны.** Иерархия
  `Country → City → Place/Event/Guide`, у City есть свои координаты, зум и
  часовой пояс — можно добавить второй город одной формой в админке.
- **«Живой город».** У `Event` есть `startAt`/`endAt`; API считает
  `isLiveNow` / `startsSoon` на лету (`src/features/events/utils.ts`), и карта
  показывает пульсирующие маркеры + отдельную ленту «Сейчас рядом» —
  это и есть главное отличие от обычной карты достопримечательностей.
- **Бронирования настоящие, не декоративные.** `/api/bookings` в транзакции
  проверяет остаток мест/билетов и атомарно списывает вместимость
  (`tourAvailability.bookedCount`, `eventTicketType.soldQuantity`,
  `guideAvailability.bookedCount`). Отмена в личном кабинете и в админке
  корректно возвращает вместимость обратно.
- **Честно про оплату.** Никакой формы «введите номер карты» — это было бы
  обманом. Бронирование подтверждается сразу (модель для MVP), а
  `Booking.paymentIntentId` — чистая точка расширения для реального шлюза
  (для Узбекистана логичнее Payme/Click, чем Stripe).
- **Изображения — свои, не внешний CDN.** `/api/placeholder/[seed]` рисует
  детерминированный SVG-градиент на лету. Ничего не задепенднено от
  Unsplash/Picsum, и в проде это тоже валидный подход для мест без реальных
  фото на старте — правки на реальные URL делаются через админку.

## Структура проекта

```
prisma/               # schema.prisma, миграции, seed.ts
src/
  app/
    (marketing)/       # публичный лендинг
    (auth)/             # sign-in / sign-up
    (app)/              # карта, поиск, места/события/гиды, личный кабинет
    admin/              # админ-панель (защищено ролью)
    api/                # Route Handlers
  features/             # бизнес-логика по модулям (map, places, events,
                         # guides, booking, favorites, reviews, search,
                         # account, admin, auth, marketing)
  components/ui/        # переиспользуемый UI-кит (shadcn-style)
  components/shared/     # навигация, тема, языковой переключатель
  lib/                   # prisma-клиент, auth-конфиг, утилиты
  i18n/                  # next-intl конфиг + словари ru/en/uz
```

## Быстрый старт (Docker)

```bash
cp .env.example .env
# сгенерируйте секрет: openssl rand -base64 32 → вставьте в AUTH_SECRET

docker compose up --build
# в отдельном терминале — миграции применяются автоматически при старте
# контейнера app; чтобы наполнить БД демо-данными:
docker compose exec app npm run db:seed
```

Приложение будет доступно на http://localhost:3000.

## Локальная разработка без Docker

Нужен Node.js 22+ и локальный PostgreSQL.

```bash
npm install
cp .env.example .env       # укажите свой DATABASE_URL
npm run db:migrate         # применить миграции
npm run db:seed            # наполнить тестовыми данными Самарканда
npm run dev
```

## Переменные окружения

Смотрите `.env.example` — там всё задокументировано. Кратко:

| Переменная | Обязательна | Назначение |
|---|---|---|
| `DATABASE_URL` | да | строка подключения PostgreSQL |
| `AUTH_SECRET` | да | секрет для подписи JWT (Auth.js) |
| `AUTH_GOOGLE_ID` / `AUTH_GOOGLE_SECRET` | нет | Google OAuth; без них кнопка Google просто не подключена |
| `AUTH_APPLE_ID` / `AUTH_APPLE_SECRET` | нет | Apple Sign In; `AUTH_APPLE_SECRET` — это подписанный JWT, генерируется отдельным скриптом Apple |
| `NEXT_PUBLIC_MAPBOX_TOKEN` | нет | без токена карта показывает аккуратный fallback вместо интерактивной карты |

## Известные ограничения V1 (осознанный выбор объёма)

- **Оплата не подключена.** Бронирование подтверждается сразу без списания
  денег — см. «Честно про оплату» выше. Точка расширения — `paymentIntentId`.
- **Расписание гидов и экскурсий редактируется только через сид/БД.**
  Админка управляет самими гидами/экскурсиями; управление слотами
  доступности (`GuideAvailability`, `TourAvailability`) — следующий шаг.
- **i18n покрывает ключевые экраны** (лендинг, навигация, переключатель
  языка/темы), но не переводит весь пользовательский контент автоматически —
  контент мест/событий уже хранится в трёх языках в БД (`nameRu/En/Uz` и т.д.),
  осталось прокинуть выбранную локаль в рендер карточек.
- **Apple Sign In** требует реальных credentials от Apple Developer Program —
  код готов, но не протестирован с реальным аккаунтом.
- **Загрузка файлов не реализована** (аватар/фото задаются ссылкой) — нет
  подключённого объектного хранилища (S3/Cloudinary). Ссылки поддерживают как
  внешние URL, так и локальные `/api/placeholder/...`.
